# rflux 对标工具功能差距评估

日期：2026-06-19

## 概述

本文档评估 rflux 作为独立 SFQ EDA 后端工具，对标 Quaigh、Yosys、JoSIM、OpenROAD/nextpnr、OpenSTA 等工具的功能差距。基于项目已有对齐文档（`quaigh-alignment.md`、`yosys-alignment.md`、`josim-parity.md`、`known-limitations.md`）和最新实现状态。

---

## 1. 综合层（对标 Quaigh + Yosys）

| 能力 | 当前状态 | 差距 |
|------|---------|------|
| 布尔优化 | **强** — 65 个 Quaigh fixture 对齐 | ✅ 基本完成 |
| AOI/OAI 门降低 | **强** — 44 种组合覆盖 | ✅ |
| DFFE 重建 | **partial** — 仅单寄存器反馈模式 | ⚠️ 需扩展到更广泛的顺序 motif |
| 等价检查 | **强** — 组合 + 单步顺序 SAT | ⚠️ 缺完整顺序等价证明 |
| Verilog 前端 | **缺失** | ❌ 仅有 IR JSON + .bench 子集 |
| BLIF/EDIF 导入 | **缺失** | ❌ 无通用网表交换格式 |
| 技术映射 | **minimal** — 最小单元库子集 | ⚠️ 缺成熟商用标准单元映射 |

**主要差距**：无 Verilog/BLIF 前端，技术映射覆盖面窄。

**优先级**：Verilog 前端是最大用户获取障碍——无 Verilog 就无法接入主流 EDA 流程。

---

## 2. 物理实现层（对标 OpenROAD / nextpnr）

| 能力 | 当前状态 | 差距 |
|------|---------|------|
| 布局 | **原型** — 层级化 + 分区 | ⚠️ 缺模拟退火/力导向等成熟算法 |
| 布线 | **原型** — A* 迷宫 + JTL/PTL | ⚠️ 缺多层详细布线、时钟树布线 |
| 拥塞建模 | **基础** — 边使用计数 | ⚠️ 缺溢出分析、热点图 |
| DRC/LVS | **缺失** | ❌ 仅有结构性检查（PTL 禁止长度等） |
| 寄生提取 | **缺失** | ❌ 无从 layout 提取 R/C/L 的能力 |
| CTS（时钟树综合） | **基础** — phase 分配 | ⚠️ 缺 buffer 插入、skew 优化 |
| GDS 完整性 | **基础** — 基本矩形导出 | ⚠️ 缺 polygon、label hierarchy |

**主要差距**：无 DRC/LVS、无寄生提取、P&R 算法为原型级。

**优先级**：DRC/LVS 和寄生提取是物理签核必需。

---

## 3. 时序分析层（对标 OpenSTA）

| 能力 | 当前状态 | 差距 |
|------|---------|------|
| 确定性 STA | **已实现** | ✅ |
| SSTA | **已实现** — sigma 模型 | ⚠️ 缺蒙特卡洛验证、相关性建模 |
| 裕度分析 | **已实现** — MC + 边界扫描 | ✅ 本次新增 |
| 多角分析 | **已实现** | ✅ |
| waveform-aware 时序 | **缺失** | ❌ SFQ 脉冲波形对时序的影响未建模 |
| 串扰延迟 | **已实现** — CouplingMap | ⚠️ 系数为 placeholder，需 JoSIM 校准 |
| OCV/AOCV derating | **缺失** | ❌ 无 on-chip variation derating |

**主要差距**：缺 waveform-aware 时序、OCV derating、串扰系数需校准。

**优先级**：waveform-aware 时序是 SFQ 特有需求，传统 STA 模型对脉冲电路不够准确。

| 能力 | 当前状态 | 差距 |
|------|---------|------|
| 内部 transient R/L/C | **已实现** | ✅ |
| JJ 非线性 | **已实现** — RCSJ + 多谐波 CPR | ⚠️ 高阶 CPR 语义仍未完整 |
| 传输线 | **已实现** — 延迟 T 模型 | ⚠️ 损耗模型仍有限 |
| 互感 | **已实现** | ✅ |

| 外部 JoSIM 对接 | **已实现** — 105 个基准 deck | ✅ |

**主要差距**：原生仿真器仍为受限子集，完整 SPICE 兼容性未达到。

**优先级**：串扰系数 JoSIM 校准和 Linux CI baseline 是近期可完成项。

## 8. 第二阶段路线图（已完成）

| 优先级 | 功能 | 影响范围 | 复杂度 | 状态 |
|--------|------|---------|--------|------|
| S1 | Verilog Phase 2（行为级 RTL） | 主流 RTL 设计入口 | 高 | ✅ 完成 |
| S1 | 布局算法升级（模拟退火） | 物理设计质量 | 高 | ✅ 完成 |
| S1 | 时钟树综合增强 | 时钟质量 | 中 | ✅ 完成 |
| S2 | 串扰感知布线 | 信号完整性 | 中 | ✅ 完成 |
| S2 | 功耗分析 | 功耗评估 | 中 | ✅ 完成 |
| S2 | 良率优化 | 工艺鲁棒性 | 中 | ✅ 完成 |

### S1-1: Verilog Phase 2

**目标**：支持行为级 Verilog，解锁主流 RTL 设计入口。

**范围**：
- `always @(*)` 组合逻辑块
- `always @(posedge/negedge clk)` 时序逻辑块
- 运算符：算术（+, -, *, /）、比较（==, !=, <, >）、位运算（&, |, ^, ~）、移位（<<, >>）
- 条件语句：if/else、case/casex/casez
- 块语句：begin/end
- reg 赋值：阻塞（=）和非阻塞（<=）

**依赖**：rflux-verilog crate 已有 Phase 1 基础（AST、lexer、parser、elaborator）

### S1-2: 布局算法升级

**目标**：从层级化放置升级为质量驱动的布局算法。

**范围**：
- 模拟退火（SA）布局器：随机扰动 + 代价函数（线长 + 拥塞 + 时序）
- 力导向布局：弹簧模型 + 引力/斥力迭代
- 代价函数：HPWL（半周长线长）、拥塞密度、关键路径延迟
- 与现有 LevelizedPlacer 共存，通过配置选择

### S1-3: 时钟树综合增强

**目标**：从 phase 分配升级为完整的 CTS。

**范围**：
- Buffer 插入：在时钟路径中插入 JTL buffer 平衡 skew
- Skew 优化：最小化时钟到达时间差异
- Clock gating：支持时钟门控降低功耗
- 与 waveform-aware 时序集成

### S2-1: 串扰感知布线

**目标**：将 CouplingMap 集成到布线代价函数中。

**范围**：
- 在 Dijkstra 最短路径中加入 coupling penalty
- `RoutingConfig.coupling_weight` 参数
- 布线后自动触发 CouplingMap 重新评估
- 与现有 CouplingMap → timing 集成路径协同

### S2-2: 功耗分析

**目标**：SFQ 电路功耗建模。

**范围**：
- 动态功耗：P_dynamic = N_jj × I_c × V_bias × f_clock
- AC 偏置功耗：已有 ac_bias 模型扩展
- 功耗报告：per-cell、per-net、total
- DSE 集成：功耗作为 Pareto 优化目标

### S2-3: 良率优化

**目标**：基于 margin 分析的自动良率优化。

**范围**：
- 设计中心化：自动调整参数使电路远离工艺边界
- 工艺角自动调优：在 PVT 空间中搜索最优工作点
- 与 margin crate 的 Monte Carlo 引擎集成
- DSE 集成：yield 作为 Pareto 优化目标

## 10. 总结

rflux 当前是一个**功能完备的 SFQ EDA 后端工具链**，18 个 Rust crate 覆盖完整流程。第一阶段（P0-P3）差距项已全部完成。

第二阶段重点：
1. **Verilog Phase 2**：行为级 RTL 支持，解锁主流设计入口
2. **布局算法升级**：从层级化放置升级为 SA/力导向
3. **时钟树综合增强**：buffer 插入、skew 优化
4. **串扰感知布线**：coupling 纳入布线代价
5. **功耗分析**：SFQ 动态功耗建模
6. **良率优化**：设计中心化、工艺角调优
