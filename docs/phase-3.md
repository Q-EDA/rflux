# Phase 3 - Timing and Verification

Status: prototype-complete for the current Rust/Python MVP scope.

This phase starts by turning the existing route-length heuristics into a real static timing pass.

## Implemented so far

- Added `rflux-timing`, a pure-Rust STA crate over `rflux-ir` + routed nets.
- Added minimal PDK timing models in `rflux-tech` with lookup-table driven interconnect delay and linear interpolation.
- Integrated STA into `rflux-flow` so `compile_layout` now reports:
  - worst setup slack
  - worst hold slack
  - critical path delay
  - analyzed timing arc count
  - setup and hold violation counts
- Added a standalone `analyze_timing` API in `rflux-flow` and the Python facade for direct STA access without consuming the full layout summary.
- Added node-scoped and clock-domain timing constraints so STA can override default arrival/required timing per endpoint.
- Added pin-level timing constraints so specific sink/source ports can override node-level timing requirements.
- Added explicit cross-domain timing constraints for `false_path`, `max_delay`, and `multicycle` checks between clock domains.
- Added explicit `false_path` reporting in STA summaries so skipped crossing arcs remain visible through `false_path_arcs` instead of disappearing into setup counts.
- Added simplified per-arc timing records to `analyze_timing` so Python callers can inspect `from/to`, false-path tagging, route length, clock-domain context, and setup/hold slack without consuming the full internal timing graph.
- Added a `verify_layout` API in `rflux-flow` and Python for:
  - macro-boundary PTL checks
  - PTL forbidden-length checks
  - structural route consistency checks
  - an event-driven simulation summary plus an optional external simulator hook
  - autogenerated transient simulation deck emission and external result parsing for events, result status, waveform path, reported violations, reported worst delay, and enumerated delay/violation details with endpoint context
- Kept the existing hold-fix reroute path opt-in through `min_hold_jtl_length_um`, but now evaluate its effectiveness using STA results.
- Exposed the new timing summary through PyO3 and the Python facade.

## Deferred follow-ons beyond the current prototype scope

- richer setup/hold constraints per port and per clock domain
- waveform-aware SFQ timing semantics beyond the current scalar delay model
- deeper hybrid event/SPICE integration beyond the current deck + external hook and scalar result parsing
- richer macro-boundary PTL electrical checks than the current structural rule set
- benchmark comparison against qPALACE / ColdFlux
