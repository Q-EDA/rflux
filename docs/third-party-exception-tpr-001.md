# rflux 第三方组件例外记录 TPR-001 v0

## 1. 基本信息

- 记录 ID：`TPR-001-A1`
- 组件名称：`josim` / `josim.exe`
- 组件类型：外部命令
- 当前版本：未在仓库内固定，按调用环境提供
- 风险级别：R3
- 对应台账项：`TPR-001`

## 2. 触发原因

- 触发时间：2026-05-22
- 触发事件：外部工具执行边界审查
- 问题摘要：当前仓库仍保留 `external_josim` 外部仿真路径。虽然已经收紧到裸命令 allowlist、仓库控制变量隔离和独立临时运行目录，但仍未具备 sandbox、签名校验和完整的输出文件生命周期约束。

## 3. 当前证据

- 自动化输出：`rflux-sim` 已有针对 allowlist、环境变量隔离、独立临时运行目录，以及成功后输出副本与运行目录清理、失败保留的回归测试。
- 人工核对来源：`docs/external-command-policy.md`、`docs/security-compliance.md`、`docs/third-party-risk-register.md`。
- 当前已知限制：外部仿真路径仍属于受信任操作者接口；不应作为不受控用户输入面。

## 4. 例外结论

- 结论类型：临时例外
- 结论摘要：在当前阶段，允许继续保留 `josim` 外部仿真路径，但仅限既有受限边界内使用。
- 当前可接受范围：
  - 仅允许裸命令 `josim` / `josim.exe`
  - 不通过 shell 拼接命令行
  - 子进程移除 `RFLOW_*` / `JOSIM_*` 环境变量
  - 输入 deck 收敛到独立 `rflux-ext-*` 临时目录
  - 成功运行后把 deck / waveform 复制到稳定临时文件，并清理原运行目录
  - 失败运行保留原运行目录，以便复审后决定是否清理
- 不可接受范围：
  - 放宽到任意可执行文件
  - 在未更新策略文件和回归测试的情况下扩展外部工具支持
  - 将该路径表述为适用于不受控输入的安全接口

## 5. 审批与责任

- DRI：核心内核负责人

- 审批时间：2026-05-22
- 有效期或复审时间：在引入新外部仿真器、放宽边界，或准备正式商业发布前复审
