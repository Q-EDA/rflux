# rflux 第三方组件例外与人工补审模板 v0

## 1. 使用说明

本模板用于记录以下两类场景：

- 第三方组件风险例外审批
- 自动化结果不足时的人工补审结论

适用对象包括但不限于：

- `docs/third-party-risk-register.md` 中登记的开放项
- Python 许可证导出中的缺失项
- 外部命令执行边界中的临时放宽或例外接受

## 3. 示例场景

### 3.1 Python 许可证缺失项

可用于 `TPR-002` 下某个包缺少 `License-Expression` / `License` / license classifiers 时的人工补审。

建议至少补充：

- wheel 或 sdist 的上游来源
- 项目主页或源码仓库中的许可证声明
- 审核人对“当前是否允许继续使用”的结论

### 3.2 外部命令例外项

可用于 `TPR-001` 下记录某次临时接受外部工具执行边界的原因和范围。

建议至少补充：

- 为什么当前不能进一步收紧
- 当前接受的附加风险
- 计划何时回到正式边界
